# Web 与 Qt 功能对齐状态

> 更新日期：2026-07-13
> 数据口径：Web 固定使用项目数据库线路，不提供“演示数据 / 数据库线路”切换。

## 已接入 Web 的主要功能

| 功能 | 当前实现 |
|---|---|
| 驾驶级位 | P1 / P2 / P3、惰行、B1 / B2 / 紧急制动均通过真实驾驶 API 生效 |
| 驾驶模式 | 支持手动 / 自动切换，并由后端统一处理控制状态 |
| 车门 | 支持左开门、右开门和关门；联锁拒绝原因由接口返回 |
| 列车设置 | 支持 AW0-AW3 载荷、5-120 秒停站时间和预设编组实时修改 |
| 调试与快进 | 支持跳转到指定车站、1 / 2 / 5 / 10 倍速及快进到下一站 |
| 力学分析 | 展示速度、限速、坡度、最大车钩力历史曲线及逐车力分量 |
| 能耗分析 | 展示牵引、再生、辅助和净功率历史，以及各类累计能量与再生率 |
| 线路数据 | WebSocket 快照包含 Seg、坡度、限速和车钩力等实时数据 |
| 运行评价 | 支持安全、准点、平稳、节能、操作规范的多维评分和综合等级 |
| PLC 输出 | Web 可逐灯控制独立 PLC 硬件灯，并可下发 ATP 允许速度 |

## 有意保留的差异

- Web 只使用数据库线路。数据源切换已按项目要求移出范围。
- 停车精度只有在后端存在可靠停车误差记录时才评分；当前无数据时显示“未采集”，且不计入综合分。
- Web 编组编辑提供常用预设；Qt 的逐车 M/T 与完整车辆参数编辑仍属于桌面端高级配置能力。
- Web 的历史曲线保留最近 120 个实时采样点，刷新页面后重新积累，不作为长期历史数据库。
