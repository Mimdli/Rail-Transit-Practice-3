# 轨道交通模拟系统 — 中期答辩报告

## 一、项目概述

本项目是基于 **Python + PyQt5** 的**轨道交通列车运行模拟系统**，旨在模拟地铁/轻轨列车在线路上的完整运行过程。项目以软件暑期实训为背景，从零构建了一个包含车辆动力学、线路数据管理、信号系统、供电系统、车门联锁、自动驾驶、运行评价等多个子系统的综合仿真平台。

### 技术栈

| 技术 | 用途 |
|------|------|
| Python 3.8+ | 主编程语言 |
| PyQt5 | 桌面 GUI（仪表盘、控制面板、线路可视化） |
| xlrd / openpyxl | 读取 Excel 格式的原始线路数据 |
| SQLite | 存储和管理线路数据 |
| pytest | 单元测试（18 个测试文件，245+ 个测试用例） |

### 2.2 自动驾驶控制器

#### 完整的停站状态机

AutoDriveController 实现了四阶段停站状态机，支持自动开关门和停站计时：

```
  CRUISING ──→ APPROACHING ──→ DWELL ──→ DEPARTING ──→ CRUISING
  (巡航)        (制动接近)      (停站)     (发车)        (巡航)
```

| 阶段 | 行为 |
|------|------|
| **CRUISING** | P 控制器维持巡航速度（目标 = 90% 限速） |
| **APPROACHING** | 前馈 + 反馈制动控制器实现精确停车 |
| **DWELL** | 自动开门 → 停站计时（默认 20s）→ 自动关门 |
| **DEPARTING** | 查找下一站 → 设为目标 → 发车 |

#### 前馈 + 反馈制动控制（新增）

制动段从纯 P 控制器升级为前馈 + 反馈联合控制：

| 控制方式 | 公式 | 作用 |
|----------|------|------|
| **前馈** | a_required = v²/(2d), feed_forward = a_required / a_available | 运动学预判所需制动级位 |
| **反馈** | feedback = Kp × (v_current - v_target) | P 控制器修正速度跟踪误差 |
| **合成** | brake_cmd = clamp(feed_forward + feedback, 0, 1) | 前馈为主，反馈为辅 |

关键参数：
- `KP_BRAKE = 1.5`：制动反馈增益
- `KP_CRUISE = 0.3`：巡航牵引增益
- `BRAKE_MARGIN = 1.3`（30% 安全余量）

#### 制动曲线计算

`_compute_brake_deceleration()` 方法根据当前编组动态计算可用制动减速度：
- 使用 FULL_BRAKE 级位（0.7）作为基准
- 包含载荷补偿（重载列车制动需求更大）
- 计入基本阻力的贡献（+0.03 m/s²）
- 施加 15% 安全系数

代码位置：[auto_drive.py](file:///e:/2025py/shixun3/Rail-Transit-Practice-3/src/vehicle/auto_drive.py)

## 三、关键技术实现

### 3.1 多体动力学仿真管线

设计目标：在保持实时性（30fps）的同时，精确模拟 6 节编组列车中每节车的独立动力学行为。

**关键设计决策**：
- **物理-UI 时钟解耦**：外部 UI 刷新间隔 33ms，内部仿真微步长 ≤1ms
- **七步闭环**：每个微步内完成"查线路 → 算力 → 算车钩 → 聚合 → 积分 → 裁剪"闭环
- **PT1 滤波器**：模拟牵引电机建磁迟滞和制动气缸充气迟滞
- **弹性车钩**：弹簧-阻尼模型，支持车钩间隙（slack）和预紧

### 3.2 线路坐标构建

核心挑战：Excel 原始数据中的 Seg 表仅存储邻居关系（正向/侧向），需要通过图遍历计算每个区段的绝对坐标。

**算法流程**：
1. 找出所有根段（不被任何段引为 end_neighbor 的段）
2. 从根段出发 BFS，仅沿正向（end_neighbor / end_lateral）推进
3. 子段坐标 = 父段.abs_start + 父段.length
4. 映射限速/坡度/信号到绝对位置

### 3.3 进路系统

- **自动模式**：沿主线 forward 链动态算路
- **预定义进路**：支持主线全程和侧线进路的预设
- **RouteManager**：进路管理独立模块，与自动驾驶控制逻辑解耦 ★新增
- **道岔消歧**：在道岔重叠区域通过活动进路选择正确的 segment

### 3.4 前馈 + 反馈制动控制 ★新增

原自动驾驶采用纯 P 控制器（仅根据速度误差输出制动指令），升级为前馈 + 反馈联合控制：

- **前馈路径**：根据运动学公式 v² = 2ad 预判所需制动级位，快速响应距离变化
- **反馈路径**：P 控制器修正速度跟踪误差，消除前馈的模型误差
- **优势**：制动响应更快，停车精度更高，对坡度变化等干扰的鲁棒性更强
