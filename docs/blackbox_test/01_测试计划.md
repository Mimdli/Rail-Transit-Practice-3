## 1. 测试目的

在不关注内部代码结构的前提下，验证系统对外可操作的功能是否满足需求规格与实训演示目标，发现接口/界面/业务规则层面的缺陷。

本计划对应队友布置任务：**按软件测试课方法组织用例与文档，每条用例带截图**。

## 2. 测试范围

### 2.1 纳入范围（In Scope）

| 编号 | 模块 | 说明 |
|------|------|------|
| M1 | 桌面启动与主界面 | 启动、Tab、数据源状态 |
| M2 | 手动驾驶与驻车 | 牵引/制动、无牵引不溜车 |
| M3 | 车门联锁 | 运行中禁止开门、开门禁牵 |
| M4 | 调度中心 | 加车、发车、扣车、紧急停车 |
| M5 | 岔口进路 UI | 走侧线 / 恢复主线 |
| M6 | 线路可视化 | 锁定视角、列车显示 |
| M7 | Web ATS | 总览、调度、场景、接口页、紧急确认 |
| M8 | API 健康 | `/api/health`、`/api/snapshot` |

### 2.2 不纳入范围（Out of Scope）

- 单元测试/白盒覆盖率（已有 `tests/`）
- 真实 704 硬件联调稳定性（需实验室环境）
- 性能压测、安全渗透

## 3. 测试环境

| 项 | 要求 |
|----|------|
| OS | Windows 10/11 |
| Python | 3.9+（推荐 3.10–3.13） |
| 依赖 | `pip install -r requirements.txt`；Web 需 `uvicorn`、`fastapi` |
| 桌面启动 | `set PYTHONPATH=.` → `python src\main.py` |
| Web 启动 | `python -m uvicorn src.web.app:app --host 127.0.0.1 --port 8765` |
| 浏览器 | Chrome / Edge |
| 截图目录 | `docs/blackbox_test/screenshots/` |

## 4. 黑盒设计方法说明（课上对应）

| 方法 | 本计划中的应用 |
|------|----------------|
| **等价类划分** | 驾驶指令（牵引/惰行/制动）、供电状态、数据源类型 |
| **边界值分析** | 速度≈0 驻车、线路总长末端里程、倍速 1/2/5/10 |
| **场景法** | 「加车→设交路→发车→停站」主成功场景及异常分支 |
| **判定表** | 车门联锁：速度×门状态×发车请求 |
| **错误推测** | 无牵引溜车、岔口卡死、紧急停车未确认取消等历史缺陷回归 |

## 5. 通过准则

- 关键用例执行率 100%
- 阻塞级缺陷 = 0
- 主要功能用例通过率 ≥ 90%
- 每条用例附带至少 1 张截图（`screenshots/TC-xxx_*.png`）

## 6. 风险

| 风险 | 缓解 |
|------|------|
| 本机无 PyQt 显示环境截图失败 | 使用 `scripts/capture_blackbox_screenshots.py`；必要时手工补屏 |
| Web 端口占用 | 改用 8766 或结束旧 uvicorn |
| 实验室网络不可达 | 联调用例标为「条件执行 / 阻塞」 |
