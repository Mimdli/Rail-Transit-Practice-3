# Rail-Transit-Practice-3 — 轨道交通模拟系统

软件暑期实训3 —— 基于 Python + Qt 的轨道交通运行模拟系统。

## 项目简介

本系统面向轨道交通运行模拟，目标是在有限实训周期内实现一个**可演示、可交互、可扩展**的轨道交通模拟系统。

系统模拟列车在线路上的基本运行过程，涵盖车辆动力学、线路条件、信号约束、供电状态、车门联锁和运行日志等核心要素，可用于手动/自动驾驶演示、安全逻辑验证及运行评价。

## 技术栈

| 技术 | 用途 |
|------|------|
| **Python 3.8+** | 主开发语言 |
| **PyQt5 / PySide6** | 桌面 GUI 框架，实现可视化界面与交互控制 |

### 1. 车辆仿真模块

模拟列车的运动状态和控制响应，采用简化动力学模型，体现：
- **牵引/制动/惰行**三种控制状态
- 上坡加速能力下降、下坡制动距离变长
- 速度越高空气阻力越明显
- 支持手动驾驶和自动运行两种模式

### 2. 轨道线路模块

从 Excel 线路数据文件中读取线路基础数据，提供：
- 车站、站台位置与方向
- 区间距离计算
- 动态限速与坡度查询
- 到站站台侧判断

### 3. 信号系统模块

模拟基本信号安全约束：
- **绿灯** — 允许正常运行
- **黄灯** — 允许运行，需降低速度
- **红灯** — 禁止越过，接近时自动制动
- 发车许可与停车防护判断

### 4. 供电状态模块

模拟供电条件对牵引能力的影响：
- 供电正常 / 电压低（限制牵引）/ 断电（禁止牵引）/ 故障恢复

### 5. 可视化展示模块

基于 Qt 的图形界面，实时展示：
- 当前速度、位置、限速、坡度
- 信号状态、供电状态、车门状态
- 当前站/下一站、下一站距离
- 运行日志与安全事件统计

### 6. 日志与运行评价模块

记录运行全过程并输出评价指标：
- 发车/到站事件、停车误差
- 超速事件、红灯违规、紧急制动
- 供电异常记录
- 评价指标：停车误差、最大速度、超速次数、安全事件数

## 需求优先级

| 优先级 | 内容 |
|--------|------|
| **Must** | 车辆运行、线路站点、限速、坡度、手动控制、状态显示、日志记录 |
| **Should** | 自动停站、信号约束、供电异常、车门联锁 |
| **Could** | 天气影响、载荷模式、黏着控制、防空转/防滑 |
| **Won't** | 完整 CBTC、完整联锁表、真实供电网络、复杂列车编组 |

## MVP 定义

> 一列车在真实或简化线路数据上运行，用户可以控制牵引、制动、惰行和车门；系统根据当前位置读取限速、坡度、站点和信号状态，并对超速、红灯、供电异常、车门未关等情况进行约束和日志记录。

## 后续扩展方向

- **天气与黏着条件**：晴天/雨天/冰雪切换，轮轨黏着系数限制
- **载荷与车辆质量**：AW0（空载）/ AW2（定员）/ AW3（超员）模式
- **防空转/防滑**：轮轨保护逻辑
